# Dissect the Incumbent Before Chasing It

**Extracted:** 2026-06-21
**Context:** 既知の定量目標（リーダーボード首位・競合ベンチマーク・SOTA スコア）に届かず「根本的に別のアプローチが要るのでは」と感じたとき。

## Problem
目標に届かないと「フレームごと変える必要がある」と思い込み、別アプローチを一から探して時間を溶かす。だが gap の正体は地味な1成分（表現の取り方・許容差の悪用・計算量）であることが多く、パラダイム転換は不要なのが大半。

## Solution
最適化に着手する**前に**、首位の実物（artifact）を解剖する:

1. **スコアでなく実物を取得** — read-only でトップ解そのもの（構成・係数・支持集合）を入手。スコアだけ見て満足しない。
2. **厳密スコアラで再現し gaming を検出** — 公式スコアラを手元で **exact**（サンプリングでなく全数）再走させ、首位が「正直」か「許容差・サンプリング・seed を突いている」かを判定。exact 検査は MC より厳しいので緩みの使用が露見する。
3. **gap を分解** — 自分と首位の差を成分に割る: 構造/表現（support・basis の選び方）/ 許容差・gaming 税 / 計算量・予算。
4. **税を分離する操作** — 首位の解を「正直化」してスコア低下を測る（制約厳格化／許容内へスケール等）。残差が真の構造差、落ちた分が gaming 税。
5. **本当のレバーを1つ叩く** — 多くの場合、別フレームでなく分解した1成分（表現を賢くする／自分も許容差を使う／計算を速くする）が効く。

## When to Use
- 既知トップに届かず「別の根本アプローチが要る?」と問いたくなったとき。
- 「物量/規模を増やせば届く」フレーム自体に違和感があるとき（規模でなく構造/税が真因のことが多い）。
- 提出/公開前に、自解が首位に対し本当に original か単なる再現かを定量化したいとき。

## Anti-pattern
スコアだけ見て「もっと最適化」or「別アプローチ探索」に直行する。実物を1回解剖すれば、grind すべき方向が偵察一発で確定する。
